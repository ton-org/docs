<Aside type="note">
  This guide applies to `@ton-pay/ui-react` `0.3.2` and lower.
</Aside>

## Install the packages

```bash
npm install @ton-pay/ui-react@0.3.2 @tonconnect/ui-react @ton-pay/api
```

If the application is pinned to an earlier `0.3.x` release, keep the matching
version. The examples below use `0.3.2`.

## 1. Configure TON Connect

React integrations in version `0.3.2` and lower depend on [TON Connect](/ecosystem/ton-connect/overview)
for wallet selection and transaction submission.

Create a TON Connect manifest and wrap the application with
`TonConnectUIProvider`.

```tsx
import { TonConnectUIProvider } from "@tonconnect/ui-react";
import { CheckoutPage } from "./checkout-page";

export default function App() {
  return (
    <TonConnectUIProvider manifestUrl="https://myapp.com/tonconnect-manifest.json">
      <CheckoutPage />
    </TonConnectUIProvider>
  );
}
```

## 2. Create the transfer on the server

The recommended flow creates the TON Pay transfer on the backend and
persists tracking identifiers before the wallet opens.

```ts
import { TON, createTonPayTransfer } from "@ton-pay/api";

const options = {
  chain: "testnet",
} as const;

app.post("/api/create-payment", async (req, res) => {
  const { orderId, senderAddr } = req.body;

  const { message, reference, bodyBase64Hash } = await createTonPayTransfer(
    {
      amount: 12.34,
      asset: TON,
      recipientAddr: "<RECIPIENT_ADDRESS>",
      senderAddr,
      commentToRecipient: `Order ${orderId}`,
    },
    options,
  );

  await db.savePayment({
    bodyBase64Hash,
    orderId,
    reference,
  });

  res.json({ bodyBase64Hash, message, reference });
});
```

## 3. Send the payment from React

`useTonPay()` receives the wallet address, requests a TON Connect message from
the factory function, and submits that message through TON Connect.

```tsx
import { useState } from "react";
import { TonPayButton, useTonPay } from "@ton-pay/ui-react";

export function CheckoutButton({ orderId }: { orderId: string }) {
  const { pay } = useTonPay();
  const [isLoading, setIsLoading] = useState(false);

  async function handlePay() {
    setIsLoading(true);

    try {
      const { reference } = await pay(async (senderAddr: string) => {
        const response = await fetch("/api/create-payment", {
          method: "POST",
          headers: { "Content-Type": "application/json" },
          body: JSON.stringify({ orderId, senderAddr }),
        });

        return response.json();
      });

      console.log("Transfer submitted", reference);
    } finally {
      setIsLoading(false);
    }
  }

  return (
    <TonPayButton
      handlePay={handlePay}
      isLoading={isLoading}
      loadingText="Processing payment..."
    />
  );
}
```

## 4. Track the result

Wallet approval is not the final payment confirmation. Keep the backend-created
identifiers and confirm the result after the transaction is submitted.

- Persist `reference` and `bodyBase64Hash` before returning the message to the client.
- Use `getTonPayTransferByReference()` or `getTonPayTransferByBodyHash()` when the UI must poll for completion.

```ts
import { getTonPayTransferByReference } from "@ton-pay/api";

const transfer = await getTonPayTransferByReference(reference, {
  chain: "testnet",
});

if (transfer.status === "success") {
  await db.markOrderAsPaid(reference, transfer.txHash);
}
```

## UI React API reference

### Imports

```ts
import { TonPayButton, useTonPay } from "@ton-pay/ui-react";
```

The React package surface is centered on those two exports. Backend
helpers remain in `@ton-pay/api`.

### `useTonPay(options?)`

`useTonPay` is the React hook that integrates with TON Connect and returns the
`pay(getMessage)` action.

#### How `pay(getMessage)` works

1. If no wallet is connected, the hook opens the TON Connect modal.
1. The hook calls the app-provided `getMessage(senderAddr)` factory.
1. The returned TON Connect message is submitted for wallet approval and signing.
1. The hook resolves with the TON Connect send result and any tracking fields returned by the factory.

```tsx
const { pay } = useTonPay();

const result = await pay(async (senderAddr: string) => {
  return {
    bodyBase64Hash: "<BODY_HASH>",
    message: {
      address: "<RECIPIENT_ADDRESS>",
      amount: "12300000000",
      payload: "<BASE64_PAYLOAD>",
    },
    reference: "<REFERENCE>",
  };
});
```

#### Return fields from `pay()`

| Field | Type | Description |
| --- | --- | --- |
| `txResult` | `SendTransactionResponse` | TON Connect transaction result with the signed transaction payload. |
| `message` | `TonPayMessage` | The TON Connect message that was sent to the wallet. |
| `reference` | `string` | Optional transfer reference returned by the message factory. |
| `bodyBase64Hash` | `string` | Optional Base64 body hash returned by the message factory. |

### `TonPayButton`

`TonPayButton` is the prebuilt React button that wraps wallet
connection and invokes `handlePay`.

#### Full props reference

All props are optional except `handlePay`.

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `handlePay` | `() => Promise<void>` | required | Payment handler called when the button is selected. |
| `isLoading` | `boolean` | `false` | Shows a loading spinner and disables the button. |
| `variant` | `"long" \| "short"` | `"long"` | Button text variant: "Pay with TON Pay" or "TON Pay". |
| `preset` | `"default" \| "gradient"` | - | Predefined theme preset. Overrides `bgColor` and `textColor`. |
| `onError` | `(error: unknown) => void` | - | Called when `handlePay` throws. |
| `showErrorNotification` | `boolean` | `true` | Shows the built-in error popup on error. |
| `bgColor` | `string` | `"#0098EA"` | Background color or CSS gradient. |
| `textColor` | `string` | `"#FFFFFF"` | Text and icon color. |
| `borderRadius` | `number \| string` | `8` | Border radius in pixels or CSS units. |
| `fontFamily` | `string` | `"inherit"` | Font family for button text. |
| `width` | `number \| string` | `300` | Button width in pixels or CSS units. |
| `height` | `number \| string` | `44` | Button height in pixels or CSS units. |
| `loadingText` | `string` | `"Processing..."` | Text shown during loading state. |
| `showMenu` | `boolean` | `true` | Shows the dropdown menu with wallet actions. |
| `disabled` | `boolean` | `false` | Disables the button. |
| `style` | `Record<string, any>` | - | Additional inline styles. |
| `className` | `string` | - | Additional CSS class name. |

#### Button behavior

- `TonPayButton` can show a built-in error popup when `handlePay` throws.
- Use `onError` for analytics or custom notifications.
- Set `showErrorNotification={false}` to suppress the built-in popup.

### On-ramp props


| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `isOnRampAvailable` | `boolean` | `true` | Show or hide the card top-up option. |
| `userIp` | `string` | - | Optional user IP passed for MoonPay geographic checks. |
