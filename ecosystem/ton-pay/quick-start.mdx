## First widget

<Steps>
  <Step
    title="Install the UI package"
  >
    Install `@ton-pay/ui-react`.

    ```bash
    npm install @ton-pay/ui-react
    ```

    <Aside type="tip">
      `TonPayWidget` does not require `@ton-pay/api`. Install it separately only
      for server-side helpers or for flows that do not use the TON Pay UI.
    </Aside>
  </Step>

  <Step
    title="Add the widget component"
  >
    Add `TonPayWidget` to the page and pass the required props.

    ```tsx
    import { TonPayWidget } from "@ton-pay/ui-react";

    <TonPayWidget
      // By default, the widget uses TON on mainnet.
      amount={12.34}
      recipientWalletAddress="<RECIPIENT_WALLET_ADDRESS>"
    />
    ```

    Required props:

    - `amount` - payment amount in asset units.
    - `recipientWalletAddress` - recipient wallet address.

    Main checkout props:

    - `asset` - exact asset identifier. Use `TON`, `USDT`, or a jetton master address.
    - `chain` - TON Pay environment. Defaults to `mainnet`.
    - `itemTitle` - UI label shown inside the widget.
    - `comment` - payment comment to a recipient that will be added to the blockchain transaction.

    `TonPayWidget` renders the button and opens the hosted TON Pay widget.
    The host app passes checkout parameters, while TON Pay handles session
    bootstrap, checkout routing, wallet flow, and result handling.

    See [API reference](/ecosystem/ton-pay/api-reference) for the full props list.
  </Step>

  <Step
    title="Run the app"
  >
    Start the React app in the usual way for the project.

    ```bash
    npm run dev
    ```

    Open the local app URL and click the button to launch the widget.
  </Step>
</Steps>

<Aside type="caution" title="Use testnet while integrating">
  The minimal example uses the default `chain="mainnet"`. During development,
  add `chain="testnet"` until the wallet, asset, and checkout flow are ready
  for production.
</Aside>

## Full example

This example scaffolds a React app with Vite, installs `@ton-pay/ui-react`, and
renders a widget with common checkout props and result callbacks.

Replace `<RECIPIENT_WALLET_ADDRESS>` with the recipient wallet address.

```bash
npm create vite@latest tonpay-widget-demo -- --template react-ts
cd tonpay-widget-demo
npm install
npm install @ton-pay/ui-react
```

```tsx
// src/App.tsx
import { TonPayWidget } from "@ton-pay/ui-react";

export default function App() {
  return (
    <main>
      <section>
        <h1>TON Pay checkout</h1>

        <TonPayWidget
          amount={12.34}
          chain="testnet"
          asset="TON"
          comment="Order #123"
          itemTitle="Electric Guitar"
          recipientWalletAddress="<RECIPIENT_WALLET_ADDRESS>"
          onError={(payload) => {
            console.error(payload.code, payload.message);
          }}
          onSuccess={(payload) => {
            console.log("Payment completed", payload);
          }}
        />
      </section>
    </main>
  );
}
```

Common optional props:

- `chain` - TON Pay environment. Defaults to `mainnet`.
- `itemTitle` - UI label shown inside the widget.
- `comment` - payment comment sent to the backend flow.
- `onSuccess` - callback fired when the widget completes successfully.
- `onError` - callback fired for widget runtime or session bootstrap errors.
