`TonPayWidget` is the React payment flow for TON Pay `0.4`. The hosted widget manages checkout state internally and returns lifecycle callbacks to the host app.

## Recommended integration

```tsx
import { TonPayWidget } from "@ton-pay/ui-react";

export function Checkout() {
  return (
    <TonPayWidget
      amount={3.5}
      chain="testnet"
      comment="Order #8451"
      currency="TON"
      itemTitle="Premium plan"
      recipientWalletAddress="<RECIPIENT_WALLET_ADDRESS>"
      onSuccess={(payload) => {
        console.log("Payment completed", payload);
      }}
      onError={(payload) => {
        console.error("Payment failed", payload);
      }}
    />
  );
}
```

## What the widget does for you

- preloads a widget session automatically
- opens the iframe checkout on click
- manages wallet connection inside the widget
- handles checkout routing and result states
- handles payment status on the TON Pay side
- forwards success and error payloads to the host app

## Success handling

```tsx
<TonPayWidget
  amount={12}
  chain="testnet"
  recipientWalletAddress="<RECIPIENT_WALLET_ADDRESS>"
  onSuccess={(payload) => {
    console.log("action:", payload.action);
    console.log("reference:", payload.reference);
    console.log("txHash:", payload.txHash);
  }}
/>
```

For direct wallet payments, `reference` is preserved in the success payload when it is available.

## Error handling

```tsx
<TonPayWidget
  amount={12}
  recipientWalletAddress="<RECIPIENT_WALLET_ADDRESS>"
  onError={(payload) => {
    console.error(payload.code, payload.message);
  }}
/>
```

Session bootstrap failures are normalized into `WidgetErrorPayload` and delivered through `onError`.

Use `comment` for the payment comment that is persisted in the backend flow.
Use `itemTitle` only for UI text shown inside the widget.

## When to use server-side integration

Use `@ton-pay/api` only when the application needs flows outside the default widget checkout, for example:

- backend transfer creation
- explicit transfer status lookups by `reference` or `bodyBase64Hash`

See [server-side integration](/ecosystem/ton-pay/payments-integration/server-side).

## Migration from version 0.3.2 or lower

The current React payment flow is centered on `TonPayWidget`, not the legacy
`TonPayButton` + `useTonPay` integration.

- Use the [migration guide](/ecosystem/ton-pay/legacy/migration) for the upgrade checklist and before/after examples.
- Use the [legacy React integration guide](/ecosystem/ton-pay/legacy/react-integration) only if the application is still pinned to `@ton-pay/ui-react@0.3.2` or lower.
