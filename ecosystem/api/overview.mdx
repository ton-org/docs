This page compares common ways to access TON data: public liteservers, hosted APIs (TON Center v2/v3, TonAPI, dTON), and self-hosted options. It focuses on whether a service can be self-hosted, is open-source, whether it indexes blockchain data, provides archival history, and whether responses come with verifiable proofs.

<sup>1</sup> Indexing means the service maintains its own database derived from blockchain data for richer queries (traces, jettons, NFTs, etc.), beyond raw lite-server RPC.

<sup>2</sup> Archival indicates historical data retention. For liteservers, this depends on the node's archival configuration; hosted indexers typically keep full history, but exact retention policies are service-specific.

<sup>3</sup> Proofs denote responses that can be verified trustlessly using cryptographic proofs from the network (liteserver/tonlib-based). HTTP indexers typically do not return proof bundles in their REST/GraphQL responses.

<sup>4</sup> TonAPI provides REST/GraphQL responses from an indexer; separate liteserver access (listed on their site) is the trustless option with proofs.
