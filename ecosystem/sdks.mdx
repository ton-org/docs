There are several ways to interact with The Open Network (TON) Blockchain:

- **HTTP** libraries connect to HTTP JSON APIs to read from and write to the blockchain. HTTP servers mostly relay these requests to the Abstract Datagram Network Layer (ADNL) network.
- **ADNL** libraries connect to a [liteserver](../node/overview#interacting-with-ton-nodes).

Protocol comparison:

|                                                      | HTTP            | ADNL            |
| ---------------------------------------------------- | --------------- | --------------- |
| Standardized                                         | No              | Yes             |
| Can connect from a web page                          | Yes             | No              |
| Has free third-party servers                         | Yes             | Yes             |
| Can be self-hosted                                   | Yes             | Yes             |
| Requires trusting third parties                      | Yes<sup>1</sup> | No              |
| First connection takes time for data synchronization | No              | Yes<sup>2</sup> |

<sup>1</sup> Some HTTP servers provide proofs, but there is no out‑of‑the‑box library that verifies them.
<sup>2</sup> If proofs returned by liteservers are ignored, the first connection skips data synchronization; however, this requires trusting the liteserver.

Software development kits (SDKs) might also provide functionality:

- **Core** libraries implement standard TON data structures (cell, slice), formats (address, mnemonic), and cryptography.
- **Wrappers** provide high-level APIs for interacting with standard contracts (wallet, jetton, and non-fungible token (NFT)).
- **Emulator** libraries provide an execution environment similar to the real blockchain for testing.
